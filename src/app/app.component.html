The Query editor/builder simple Demo for supporting auto complete

Boolean operators will work as follows:

  • Quotes: searches for mentions that meet the exact phrase
  • AND : searches for mentions of 2 keywords together on a page
  • OR: searches for mentions of 1 or the other keywords on a page
  • NOT searches for mentions of a keyword on a page and will return the page as a match if the other keyword is not mentioned on that page
  • Brackets: used to organize logic around operators and keywords for compound queries
  • NEAR/ x: search for a word or set of words within an x range of each other using the format “(a OR b OR c Or ...) NEAR/x (d OR e OR f OR ...)”
  • The native “~” (typing tilde) proximity operator would still be available for all other cases
  • And other operators such as: hashtags, continent, country, state, county, city, site, url, title, author, links and wildcard (?, *)
